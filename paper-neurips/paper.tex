% ABSTRACT
% ============================================================
\begin{abstract}
  Predicting events in multivariate time series - loss of a turbine blade, an anomalous server spike, a cardiac arrhythmia - demands methods that generalise across event types and domains, especially when labels are scarce. Existing approaches silo each event type into a separate supervised model and evaluate with incomparable, domain-specific metrics. We introduce FAM (Forecast-Anything Model), built on two ideas. First, \emph{predictor finetuning}: a causal JEPA encoder is pretrained per-dataset with no labels; downstream, we freeze the encoder and finetune only the horizon-conditioned predictor (${\sim}9\%$ of parameters) for each event type. Second, we evaluate all tasks through a probability surface $p(t, \Delta t)$ - event probability as a function of observation time and prediction horizon $\Delta t$ - parameterized as a discrete-time survival CDF that is monotonically non-decreasing by construction. Across 12 datasets in 8 domains, a single 2.16M-parameter architecture with no dataset-specific hyperparameters matches or exceeds domain-specific baselines, matching Chronos-2 on turbofan prognostics with $24\times$ fewer trainable parameters while outperforming it on the majority of benchmarks where domain-specific pretraining provides an edge. Code and stored probability surfaces at \url{https://github.com/Forgis-Research/IndustrialJEPA}.
\end{abstract}

% 1. INTRODUCTION

\section{Introduction}
\label{sec:intro}

A turbine blade cracks after 12{,}000 flight hours. A satellite sensor drifts silently for 48 hours. A patient's ECG shifts six hours before cardiac arrest. These events are rare in operational data, yet they follow partially predictable precursor dynamics~\citep{taleb2007blackswan,scheffer2009earlywarning}. Current methods treat each event type as a separate supervised problem: RUL models~\citep{fan2024star} never see anomaly benchmarks; anomaly detectors~\citep{xu2022anomalytransformer,mtsjepa2026} never predict time-to-failure.

Yet every event prediction task has the same structure: given observations up to time $t$, estimate $P(\text{event within } \Delta t)$ for each horizon $\Delta t$. The model that produces this probability surface does not need to know what kind of event it is predicting. It needs to learn the dynamics of normality; the deviation from that normality defines the event.

\paragraph{FAM.} We pretrain a causal Transformer encoder to predict future sensor representations in latent space via a Joint Embedding Predictive Architecture (JEPA)~\citep{assran2023ijepa}. The horizon-conditioned predictor $\predictor$ maps the encoder's summary $\h_t$ to a predicted future representation at variable horizon $\Delta t$, forcing the encoder to internalise dynamics at multiple timescales. Pretraining uses standard components (patches~\citep{nie2023patchtst}, instance normalisation~\citep{kim2022revin}, causal attention, EMA target~\citep{assran2023ijepa}) and is self-supervised.

\paragraph{Core idea: predictor finetuning.} After pretraining, every SSL method discards its prediction head and probes the encoder. We do the opposite: freeze the encoder, finetune the predictor. The predictor is a small horizon-conditioned MLP (${\sim}9\%$ of parameters) that, together with a per-horizon event head, is more expressive than a linear probe but far cheaper than end-to-end finetuning. The output is parameterized as a discrete-time survival CDF~\citep{lee2018deephit}, guaranteeing that event probabilities are monotonically non-decreasing in the prediction horizon by construction.

\paragraph{Contributions.}
\begin{enumerate}[nosep]
  \item \textbf{Predictor finetuning as the downstream recipe.} Freeze the encoder, finetune the predictor + head. This achieves the same AUPRC as end-to-end finetuning with $24\times$ fewer trainable parameters and tighter cross-seed variance (\cref{sec:finetune_ablation,sec:label_efficiency}).

  \item \textbf{Multi-domain event prediction under a unified metric.} One architecture across 12 datasets in 8 domains, evaluated through the same probability surface $p(t, \Delta t)$ scored by AUPRC, threshold-free and robust to class imbalance (\cref{sec:metrics,sec:experiments}).
\end{enumerate}

% 2. RELATED WORK

\section{Related Work}
\label{sec:related}

\paragraph{Self-supervised learning for time series.}
Contrastive methods (TS2Vec~\citep{yue2022ts2vec}, TNC~\citep{tonekaboni2021tnc}) learn by contrasting pairs. Masked reconstruction (PatchTST~\citep{nie2023patchtst}, SimMTM~\citep{dong2023simmtm}) reconstructs masked patches. JEPA predicts future \emph{representations}, avoiding reconstruction bias~\citep{assran2023ijepa,bardes2024vjepa}. For time series, TS-JEPA~\citep{tsjepa2024} applies temporal masking for classification; MTS-JEPA~\citep{mtsjepa2026} adds codebook regularisation for anomaly detection. All of these discard their pretraining head at inference and probe only the encoder. We keep the predictor and show it carries event-relevant signal.

\paragraph{Foundation models for time series.}
Chronos~\citep{ansari2024chronos}, TimesFM~\citep{das2024timesfm}, and MOMENT~\citep{goswami2024moment} pretrain on large-scale corpora for generic forecasting. FAM is complementary: a mid-scale encoder ($\approx$\,2.16M) specialised for event prediction, not forecasting. We benchmark against the multivariate \texttt{amazon/chronos-2} foundation model (\cref{sec:foundation_baselines}).

\paragraph{Prognostics and anomaly prediction.}
C-MAPSS~\citep{saxena2008cmapss} is the standard RUL benchmark; supervised SOTA: STAR~\citep{fan2024star} (RMSE 10.61). Anomaly prediction methods (Anomaly Transformer~\citep{xu2022anomalytransformer}, DCdetector~\citep{yang2023dcdetector}, TranAD~\citep{tuli2022tranad}) report point-adjusted F1 (PA-F1), which inflates scores by crediting entire segments from a single detection~\citep{schmidl2022tsadeval}. We replace these incomparable metrics with AUPRC pooled over the probability surface (\cref{sec:metrics}).

\paragraph{Channel strategies.}
PatchTST~\citep{nie2023patchtst} showed channel-independent processing often wins; iTransformer~\citep{liu2024itransformer} showed variate-as-token wins on high-correlation data. Our default is channel-fusion (linear projection); we ablate against sensor-as-token (\cref{sec:ablations}).

\section{Method}
\label{sec:method}

\subsection{Preprocessing}
\label{sec:preprocessing}

We follow standard practice for multi-domain time series~\citep{nie2023patchtst,ansari2024chronos,goswami2024moment}: patch tokenisation with per-window instance normalisation.

\textbf{Patches.} $P{=}16$ consecutive raw samples form one token. This is fixed for all datasets with sufficient temporal resolution (sample rate $\geq$ 1/min); for coarsely sampled data (Sepsis at 1/hour), we use $P{=}1$ so that the causal transformer receives enough tokens. \Cref{tab:tokenization} lists the resulting token counts. The encoder requires a minimum context of ${\sim}128$ timesteps (${\sim}8$ tokens) for self-attention to be effective. The context length is variable at both training and inference: for prognostic datasets with a clear lifecycle start (C-MAPSS, Sepsis), we feed the full history; for continuous monitoring streams (SMAP, PSM), we use a sliding window of 512 steps.

\textbf{Instance normalisation.} Per window, per channel: subtract the window mean, divide by the window standard deviation. This removes distribution shift across machines and operating conditions without requiring training-set statistics at inference.

\textbf{Projection.} A linear layer $\W \in \R^{d \times (C \cdot P)}$ maps each patch to a $d$-dimensional token ($d{=}256$; $C$ = number of channels, $P$ = patch length). Sinusoidal positional encodings are added.

\subsection{Architecture}
\label{sec:architecture}

\begin{figure}[t]
  \centering
  \includegraphics[width=\textwidth]{figures/fig_architecture.pdf}
  \caption{\textbf{FAM architecture.} \emph{Top:} pretraining. The context encoder $\encoder$ (causal) produces $\h_t$; the predictor $\predictor$ maps $(\h_t, \Delta t)$ to $\hat{\h}$; the EMA target encoder $\target$ (bidirectional + attention pool) provides the L1 target $\h^*$ from the cumulative interval $\x_{(t, t+\Delta t]}$ (no gap between context and target). \emph{Bottom:} finetuning. The encoder is frozen; the predictor runs at $K$ horizons; per-horizon hazard logits pass through sigmoid $\to$ conditional hazards $\lambda_k$ $\to$ cumulative product $\to$ survival CDF $p(t, \Delta t_k)$, which is monotonically non-decreasing by construction.}
  \label{fig:architecture}
\end{figure}

\textbf{Context encoder} $\encoder$: a causal Transformer ($d{=}256$, 2 layers, 4 heads). Given observations $\x_{\leq t}$, it produces a summary embedding:
\begin{equation}
  \h_t = \encoder(\x_{\leq t}) \in \R^d.
  \label{eq:encoder}
\end{equation}

\textbf{Predictor} $\predictor$: a 2-layer MLP. Given $\h_t$ and a prediction horizon $\Delta t$, it produces a predicted embedding of the future interval $(t, t{+}\Delta t]$:
\begin{equation}
  \hat{\h}_{(t,t+\Delta t]} = \predictor(\h_t, \Delta t).
  \label{eq:predictor}
\end{equation}
During pretraining, $\Delta t \sim \text{LogU}[1, {\Delta t}_\text{max}]$ is sampled randomly, forcing the encoder to internalise dynamics at multiple timescales.

\textbf{Target encoder} $\target$: an EMA copy of $\encoder$ ($\tau{=}0.99$) that encodes the next $\Delta t$ steps $\x_{(t, t+\Delta t]}$ bidirectionally with attention pooling $\to \h^*_{(t,t+\Delta t]} \in \R^d$. Crucially, the target interval starts immediately at $t$; there is no gap between context and target. The same interval is used for event prediction downstream (\cref{sec:downstream}). We also experiment with a curriculum variant that transitions from EMA to SIGReg spectral regularisation~\citep{balestriero2025lejepa} (which eliminates the target encoder entirely; see \cref{sec:ablations}).

\textbf{Pretraining loss}: $\loss = \|\hat{\h}_{(t,t+\Delta t]} - \text{sg}(\h^*_{(t,t+\Delta t]})\|_1 + \lambda \cdot \text{var-reg}$, where sg denotes stop-gradient.

\subsection{Downstream: Freeze Encoder, Finetune Predictor}
\label{sec:downstream}

Given event labels, we freeze $\encoder$ and finetune $\predictor$ + a linear head:

\begin{table}[h]
  \centering
  \caption{Finetuning modes. Predictor-FT is our default.}
  \label{tab:finetune_modes}
  \small
  \begin{tabular}{@{}lcccr@{}}
    \toprule
    Mode                  & Encoder         & Predictor          & Head            & Params tuned  \\
    \midrule
    Frozen probe          & frozen          & frozen             & LN + linear     & 513           \\
    \textbf{Predictor-FT} & \textbf{frozen} & \textbf{finetuned} & \textbf{LN + linear} & \textbf{$\approx$\,198K} \\
    E2E                   & finetuned       & finetuned          & LN + linear     & $\approx$\,2.16M         \\
    \bottomrule
  \end{tabular}
\end{table}

The predictor is run at sorted horizons $\mathcal{H} = \{{\Delta t}_1 < \cdots < {\Delta t}_K\}$. A shared linear head maps each predicted representation to a per-interval \emph{conditional hazard}:
\begin{equation}
  \lambda_k(t) \;=\; \sigma\!\bigl(\mathbf{w}^\top \hat{\h}_{(t, t+\Delta t_k]} + b\bigr) \;\in\; (0, 1),
  \label{eq:hazard}
\end{equation}
where $\lambda_k(t) = P(\text{event in } (\Delta t_{k-1}, \Delta t_k] \mid T^* > \Delta t_{k-1}, \mathbf{x}_{\leq t})$ and $T^*$ is the time-to-next-event. No distributional assumptions are required: each $\lambda_k$ is a free function of $\h_t$ via the predictor network. The event probability surface is the discrete-time survival CDF~\citep{lee2018deephit,ren2019drsa}:
\begin{equation}
  p(t, \Delta t_k) \;=\; 1 - \prod_{j=1}^{k} (1 - \lambda_j(t)).
  \label{eq:event_prob}
\end{equation}
The finetuning loss sums positive-weighted binary cross-entropy over horizons:
\begin{equation}
  \loss_\text{FT} = \sum_{k=1}^{K} w^+ \cdot \text{BCE}\bigl(p(t, \Delta t_k),\; y(t, \Delta t_k)\bigr),
  \label{eq:ft_loss}
\end{equation}
where $y(t, \Delta t) = \mathbb{1}[\text{event in } (t, t{+}\Delta t]]$ and $w^+ = N_{\text{neg}} / N_{\text{pos}}$. Two structural properties follow from the parameterization, not from post-hoc enforcement:
\begin{enumerate}[nosep,leftmargin=*]
  \item \textbf{Monotonicity by construction.} Each factor $(1 - \lambda_j) \in (0, 1)$, so the survival product $\prod_{j \leq k}(1 - \lambda_j)$ is non-increasing in $k$, and $p(t, \Delta t_k) = 1 - \prod(\cdot)$ is non-decreasing. Zero violations by design.
  \item \textbf{Interval arithmetic.} $P(\text{event in } (t{+}a,\; t{+}b]) = p(t, b) - p(t, a)$ for $b > a$, since $p$ is a valid CDF.
\end{enumerate}

\section{Evaluation}
\label{sec:metrics}

The model outputs $p(t, \Delta t)$ (\cref{eq:event_prob}) for each observation time $t$ and prediction horizon $\Delta t$. This probability surface is the complete prediction; every metric is a readout of it.

\subsection{Primary Metric: AUPRC}

Events are rare: turbine failure happens once per engine lifecycle; anomaly segments cover ${\sim}1{-}13\%$ of test data. In the probability surface $p(t, \Delta t)$, the vast majority of cells are $y{=}0$. A trivial model predicting $p{=}0$ everywhere achieves excellent Brier scores and low cross-entropy, both dominated by the majority class. We therefore use \textbf{AUPRC} (area under the precision-recall curve), which focuses entirely on the positive class and is threshold-free:

\begin{equation}
  \text{AUPRC} \;=\; \int_0^1 \text{Precision}(r) \, dr
  \label{eq:auprc}
\end{equation}

computed over all $(t, \Delta t)$ cells pooled into a single ranked list. One number per dataset. Higher is better. A random classifier achieves AUPRC equal to the prevalence (e.g., $0.01$ for $1\%$ event rate), so improvements above the base rate are meaningful.

As a secondary metric, we report \textbf{AUROC} pooled over the same cells - threshold-free discrimination that is more familiar but less sensitive to class imbalance~\citep{davis2006auprc}.

\subsection{What We Report}

\textbf{Primary:} AUPRC pooled over the full $(t, \Delta t)$ surface (\cref{eq:auprc}). \textbf{Secondary:} AUROC pooled over the same cells. \textbf{Legacy:} RMSE for C-MAPSS, PA-F1 for SMAP/MSL/PSM/SMD/MBA, derived from the stored probability surface, not computed independently. \textbf{Appendix:} AUPRC($\Delta t$) per-horizon breakdown and reliability diagrams (calibration). All results: mean $\pm$ std ($N$ seeds, 95\% CI). Claims use paired $t$-tests, Wilcoxon, Cohen's $d$. The probability surface is stored as \texttt{.npz} for every run, enabling recomputation of any metric without re-running inference.

\subsection{Additional Domains}
\label{sec:new_domains}

Three datasets expand coverage beyond the core benchmark (\cref{tab:benchmark}): GECCO 2018 (water-quality IoT, 9 channels), BATADAL (ICS cybersecurity, 43 channels), and PhysioNet 2012 (ICU mortality, 36 channels, $P{=}1$). FAM reaches within 1\,pp AUROC of the supervised InceptionTime SOTA on PhysioNet 2012 ($0.858$ vs $0.868$) with a shared architecture. On GECCO and BATADAL, FAM under-performs domain-specific models; these datasets are included for breadth, not as SOTA claims.

\subsection{Foundation Model Baselines}
\label{sec:foundation_baselines}

We compare FAM to \texttt{amazon/chronos-2}~\citep{ansari2024chronos} - a \emph{multivariate} time-series foundation model ($d_\text{model}=768$) pretrained on a heterogeneous corpus orders of magnitude larger than any single per-dataset FAM pretrain - under an identical downstream protocol: we \texttt{embed()} each observation window through the frozen Chronos-2 encoder, mean-pool over tokens to a 768-d per-observation vector, and train a linear probe on the \emph{same labels, horizons, and splits} used by FAM pred-FT, with the same positive-weighted BCE loss on the per-horizon sigmoid head. \section{Ablations}

\subsection{Finetuning Mode}
\label{sec:finetune_ablation}

\Cref{tab:finetune_ablation} isolates the paper's central claim. We compare five downstream modes on FD001 at two label budgets, all using the same pretrained backbone and the same scalar-RUL regression objective; only the set of trainable parameters changes:

\begin{enumerate}[nosep]
  \item \textbf{probe\_h}: linear probe on $\h_\text{past}$ alone (\texttt{Linear}$(d\to 1)$, 257 parameters).
  \item \textbf{frozen\_multi}: linear probe on the concatenation $[\hat{\h}_1;\ldots;\hat{\h}_{16}]$ (\texttt{Linear}$(16d\to 1)$, $\approx$\,4K parameters). Predictor frozen.
  \item \textbf{pred\_ft}: predictor finetuned (790K parameters) plus \texttt{Linear}$(16d\to 1)$; encoder frozen.
  \item \textbf{e2e}: full backbone finetuned (2.37M parameters). Our reference upper bound.
  \item \textbf{scratch}: identical arch, random init, trained end-to-end (2.37M parameters).
\end{enumerate}

\textbf{Reading the table.} Three observations survive a paired analysis across matched seeds:

\emph{(i) Pred-FT vs.\ scratch is robustly significant at 5\% labels}: paired $t(4) = 3.03$, $p = 0.039$, Cohen's $d = 1.35$. Scratch collapses to $F_w = 0.035$ while pred-FT delivers $0.261$. This $0.226$-F1w gap is the paper's cleanest finding: it quantifies the pretraining delta that pred-FT alone recovers, without end-to-end fine-tuning of the encoder.

\emph{(ii) Pred-FT vs.\ E2E at 10\% labels is statistically significant at $N=10$ (Phase 8).} At 10\% labels, pred-FT $F_w = 0.291 \pm 0.120$ vs.\ E2E $0.133 \pm 0.164$; $\Delta = +0.157$; paired $t(9) = 2.74$, $\mathbf{p = 0.023}$ (two-sided); Wilcoxon signed-rank one-sided $\mathbf{p = 0.010}$; Cohen's $d = 0.87$ (large effect). Seed-level collapses to $F_w = 0$: pred-FT $\mathbf{0/10}$, E2E $4/10$. This is the paper's statistically-supported pred-FT-beats-E2E result: 10\% labels is the label regime where E2E has enough data to sometimes learn but not enough to learn reliably, and pred-FT's frozen encoder keeps it above the floor.

\emph{At 20\% labels the crossover flips and E2E has the robustness advantage (Phase 9, $N=10$).} pred-FT $F_w = 0.276 \pm 0.184$ vs.\ E2E $0.312 \pm 0.102$, paired $t(9) = -0.59$, $p = 0.57$ (two-sided); Wilcoxon two-sided $p = 0.77$; Cohen's $d = -0.19$. Collapse count flips: pred-FT $\mathbf{2/10}$, E2E $\mathbf{0/10}$. E2E at 20\% has enough data to learn reliably every seed; pred-FT occasionally fails because freezing the encoder loses expressivity when the task has enough labels to need it. Combined with Phases 7 and 8, the empirical crossover between pred-FT and E2E sits between 10\% and 20\% labels: below that pred-FT is the safer choice; above it, E2E is.

\emph{(iii) Pred-FT vs.\ E2E at 5\% labels: robustness is the dominant signal, not mean superiority.} We also extended Phase 0b to 10 paired seeds at 5\% labels (Phase 7). At $N=10$: pred-FT $F_w = 0.229 \pm 0.141$ vs.\ E2E $0.124 \pm 0.201$, paired $t(9) = 1.75$, $p = 0.114$ (two-sided); Wilcoxon one-sided $p = 0.065$; Cohen's $d = 0.55$ (medium effect). Mean-difference tests miss $p < 0.05$. The collapse-rate difference is decisive: E2E collapses to $F_w = 0$ on \textbf{7 of 10 seeds}, pred-FT on $1/10$. At 5\% labels E2E is essentially a coin flip between ``works'' and ``fails completely''; pred-FT is monomodal with a floor near $0.23$.

\emph{(iv) The full label-efficiency curve} (\cref{tab:label_efficiency}) shows the pred-FT/E2E gap is largest at 10\% labels ($+0.177$ in favor of pred-FT, $p = 0.15$), not at 5\%. At 20\% labels E2E recovers and surpasses pred-FT (0.295 vs.\ 0.212); at 50-100\% labels they are tied. The crossover we claim is thus \emph{at $\leq 10\%$ labels} rather than a uniform low-label advantage.

\textbf{RMSE vs.\ F1w disagreement at 5\%.} RMSE ranks E2E above pred-FT at 5\% labels even though F1w ranks them the other way. The cause is scale calibration: AUROCw at 5\% is $\approx 0.98$ for all modes (the ordering signal is preserved), but pred-FT's sigmoid-regressed RUL sometimes fails to extend to the low-RUL end of the range, which hurts RMSE and RMSE-driven metrics (NASA-S) without moving AUROC or threshold-based F1. E2E's full capacity calibrates scale better.

\subsection{Does the Predictor Help at Inference?}
\label{sec:predictor_ablation}

A different question from ``does finetuning the predictor help'' is: \emph{at inference time, is the predictor's forward rollout actually informative, or does $\h_\text{past}$ already summarize everything?} Rows 1-2 of \cref{tab:finetune_ablation} answer this with \emph{all weights frozen} on both sides, isolating the predictor's inference-time contribution:

\begin{itemize}[nosep]
  \item \textbf{probe\_h} ($\h_\text{past}$ only): $0.299$ F1w at 100\%.
  \item \textbf{frozen\_multi} (linear on $[\hat{\h}_1;\ldots;\hat{\h}_{16}]$): $0.148$ F1w at 100\%.
\end{itemize}

The predictor's rollouts are \emph{not} more useful than $\h_\text{past}$ when both are frozen: frozen\_multi is \emph{worse}. We verified that the pretrained predictor's outputs have high cross-horizon cosine similarity ($>0.98$ between $\hat{\h}_1$ and $\hat{\h}_{15}$) - the horizon-conditioning signal is weak in the pretrained predictor, so the concatenation is nearly redundant and the larger input dim hurts generalization at this sample size. This is not a defect of pred-FT as a \emph{finetuning} recipe; it just means the pretrained predictor's representations, frozen, do not expose a per-horizon signal a linear head can exploit. Finetuning the predictor changes this: pred\_ft's per-horizon outputs are no longer redundant because gradients push them apart (see \cref{tab:finetune_ablation}, row 3). The ablation reinforces that pred-FT's value is in \emph{reshaping} the predictor for the task, not in extracting fixed forward rollouts.

\subsection{Pretrained Predictor vs.\ Random-Init Predictor}
\label{sec:predictor_pretrain_ablation}

A cleaner and narrower test of \emph{which part} of pred-FT carries the weight. We hold the pretrained \emph{encoder} fixed and toggle only the predictor's initialisation: (A)~inherit the predictor weights from self-supervised pretraining, (B)~re-initialise the predictor to random (same architecture, same size) and keep the encoder frozen. Both are then pred-FT'd identically. Run on FD001, SMAP, and MBA (3 seeds each):

\begin{itemize}[nosep]
  \item \textbf{FD001}: pretrained $0.9257 \pm 0.0008$ vs.\ reset $0.9235 \pm 0.0027$, paired $\Delta = +0.0021$, $t(2) = 1.12$, $p = 0.38$ (two-sided).
  \item \textbf{SMAP}: pretrained $0.3874 \pm 0.0205$ vs.\ reset $0.3950 \pm 0.0286$, paired $\Delta = -0.0076$, $t(2) = -1.24$, $p = 0.34$.
  \item \textbf{MBA}: pretrained $0.9465 \pm 0.0004$ vs.\ reset $0.9435 \pm 0.0016$, paired $\Delta = +0.0031$, $t(2) = 3.13$, $p = 0.089$.
\end{itemize}

On FD001 and SMAP the pretrained and random-init predictors are within seed noise (SMAP's reset is even marginally better). On MBA (where FAM wins the Chronos-2 comparison by the widest AUPRC margin) the pretrained predictor trends higher ($\Delta = +0.003$) but does not clear $p < 0.05$. Across three datasets the pretrained predictor weights carry at most a small advantage - the value of pred-FT is chiefly in (i) the pretrained \emph{encoder}'s representation (visible in the FAM vs Chronos-2 SMAP gap of $+0.110$ AUPRC and MBA gap of $+0.029$, \cref{sec:foundation_baselines}) and (ii) the recipe of freezing the encoder and finetuning a small 198K-parameter horizon-conditioned MLP + 513-parameter LN-linear head with pos-weighted BCE on the per-horizon logits. JEPA pretraining of the predictor is a small-to-null term; practitioners can drop it without measurable harm on FD001/SMAP, and lose at most ${\sim}0.3\%$ AUPRC on MBA.

\subsection{Causal vs.\ Bidirectional Encoder}
\label{sec:causal_ablation}

The context encoder is causal so that $\h_\text{past}$ summarises only information available before the event horizon, matching the deployment setting in predictive-maintenance and streaming-anomaly contexts. A bidirectional variant would leak information across the forecast boundary during pretraining; that said, a controlled causal-vs-bidirectional comparison is not included in this version of the paper because our v20 session budget did not include an additional bidirectional pretraining run (the other ablations reuse existing v17 checkpoints). The corresponding cells in \cref{app:decisions} are open future work.

\textbf{Pretraining-compute confound on FD003.} The two FD003 blocks in \cref{tab:label_efficiency} show opposite patterns at low labels: the 200-epoch-pretrained encoder gives poor pred-FT F1w at 5-10\% labels (0.04-0.06), while the 100-epoch ckpt gives 0.12-0.14. \begin{itemize}[nosep]
  \item \textbf{FD001 (single op.\ condition, HPC fault):} pred-FT significantly beats E2E at 10\% labels ($p = 0.023$, $N = 10$, $d = 0.87$). E2E collapses to $F_w = 0$ on 4-7 of 10 seeds at $\leq 10\%$ labels.
  \item \textbf{FD003 (single op.\ condition, HPC+fan), 200-epoch pretrain:} pred-FT significantly beats E2E at 100\% labels ($p = 0.043$, $N = 5$, $d = 1.25$). E2E collapses to $F_w < 0.03$ on 2/5 seeds even at full labels. Under the 100-epoch pretrain the direction at 100\% matches FD001 (E2E ahead, no collapses); pred-FT regains low-label advantage.

Crucially, E2E \emph{never} collapses on FD002 across the 15 runs we did - multi-condition data supplies enough supervisory signal that E2E's collapse failure mode disappears.
\end{itemize}
The unified statement is not ``pred-FT always wins at low labels.'' It is narrower and more interesting: \emph{pred-FT wins on the cells where E2E collapses}, and the collapse rate depends on both dataset and label budget. When E2E has enough diverse data to learn reliably (FD002 100\% and 10\%; FD001 and FD003 at 100\% with FD001 being a specific exception), E2E's higher capacity prevails. Pred-FT's advantage is a robustness advantage, not a mean-capacity advantage, and where E2E's floor is high the advantage narrows or inverts.

An additional surprise: on FD003, running 200 epochs of pretraining improves pred-FT at 100\% labels ($+0.124$ F1w) but \emph{reduces} pred-FT at 5-10\% labels ($-0.05$ to $-0.09$ F1w) relative to the 100-epoch ckpt. Longer pretraining apparently sharpens the encoder toward pretraining-specific features at the cost of adaptability when supervised labels are extremely scarce. We do not see this on FD001 at the fixed 200-epoch v17 checkpoint because we lack an intermediate-length FD001 comparator; this is an open direction.

\subsection{Architecture Sensitivity}
\label{sec:arch_sensitivity}

The default FAM architecture is the v24 canonical configuration ($d{=}256$, 2 layers, 4 heads, $d_{ff} = 256$, $\approx$\,2.16M parameters). The legacy V17 variant ($d_{ff} = 1024$, 2.37M parameters, 790K predictor) is retained only inside the ablation tables of \cref{sec:ablations} for historical continuity. The main finding relevant to this paper is that the predictor's cross-horizon diversity matters far more than encoder depth or width: frozen\_multi's $F_w = 0.148$ (\cref{tab:finetune_ablation}) and the high inter-horizon cosine ($>0.98$) show that the horizon-conditioning signal in the pretrained predictor is under-used unless the predictor is finetuned. SIGReg-pred pretraining increases predictor diversity explicitly via spectral regularisation and improves downstream F1w by $+0.06$ (\cref{tab:sigreg_ablation}), consistent with this view. A full depth/width sweep is out of scope for this session; the architecture-decision log in \cref{app:decisions} records the choices with their evidence source.

% ============================================================
\section{Limitations}
\label{sec:limitations}

\begin{itemize}[nosep]
  \item \textbf{Per-dataset pretraining.} We pretrain a separate encoder per dataset. Cross-domain pretraining on a union of datasets is the natural next step.
  \item \textbf{FD003 regression.} AUPRC drops to $0.766$ under the canonical $P{=}16$ architecture; the multi-fault degradation mode may need finer temporal resolution or more predictor capacity.
  \item \textbf{Chronos-2 parity on well-covered domains.} On datasets whose distribution is close to the Chronos-2 pretraining corpus, a 120M frozen encoder matches FAM. FAM's advantage is on out-of-distribution domains.
  \item \textbf{Anomaly ``prediction'' vs detection.} On SMAP, the model detects ongoing anomaly segments rather than predicting onset with lead time. True early warning requires lead-time-aware evaluation.
\end{itemize}

% ============================================================
% 8. CONCLUSION
% ============================================================
\section{Conclusion}

Time series foundation models optimize for forecasting: predicting the next values of every channel. But practitioners rarely care about the full future trajectory - they care about \emph{events}: will this turbine fail? Is this patient deteriorating? FAM inverts the priority: the encoder learns dynamics via self-supervised JEPA pretraining, then the predictor is steered toward the specific event that matters, using as few labels as needed. The probability surface $p(t, \Delta t)$, parameterized as a discrete-time survival CDF, unifies degradation, anomaly, and clinical prediction under one threshold-free metric. Across 12 datasets in 8 domains, a lean 2.16M-parameter architecture matches Chronos-2 (120M parameters, pretrained on orders of magnitude more data) and outperforms it where domain-specific dynamics matter most. The encoder learns what changes; the predictor learns what matters.

\section{PA-F1 Inflation}
\label{app:pa_f1}

PA-F1 (Point-Adjusted F1) credits an entire anomaly segment as correctly predicted if any single timestep within it exceeds the threshold~\citep{xu2022anomalytransformer}. This protocol inflates reported F1 dramatically when anomaly segments are long and prevalence is non-trivial: the TSAD-Eval study~\citep{schmidl2022tsadeval} shows that a random detector can exceed $F_1 = 0.9$ PA on some datasets. We concretely demonstrate this by contrasting PA-F1 and non-PA F1 for FAM+Mahalanobis on our benchmarks: SMAP $0.793$ PA vs.\ $0.038$ non-PA, SMD $0.252$ PA vs.\ $0.144$ non-PA, PSM $0.813$ PA vs.\ $0.085$ non-PA. The gap in PA-F1 between FAM and MTS-JEPA remains large even under a random-baseline sanity check (v18 phase 12 reports random-init + Mahalanobis at $0.604 \pm 0.007$ PA-F1 on SMAP; FAM is $0.79$), so the PA-F1 margin is not purely inflation - but any single PA-F1 number should be read in conjunction with the corresponding non-PA number.
